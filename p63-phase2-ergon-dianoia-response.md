## The Shadow: Ergon Overexpressed

**Risk 1: Function Rigidity**
- Ergon becomes enforcement: "You must only do X, never Y"
- Agents lose adaptive capacity
- Innovation requires crossing function boundaries
- **Mitigation:** Ergon tracks drift but doesn't block it. Drift is visible, not forbidden.

**Risk 2: Purity Testing**
- Every sprint analyzed for ergon alignment
- Agents second-guess every action: "Is this really my function?"
- Paralysis through over-analysis
- **Mitigation:** Ergon assesses trends (drift ratio over 10+ sprints), not individual actions

**Risk 3: Missing Cooperative Flexibility**
- Two-agent workshop requires both agents to stretch
- Six-agent workshop allows specialization
- Ergon must distinguish "necessary stretch" from "function creep"
- **Mitigation:** Ergon evaluates context. Temporary drift under capacity constraints is healthy. Permanent drift when alternatives exist is dysfunction.
