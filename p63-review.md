## Edge Cases & Questions

### 1. Formation Sequence Risk

Nou asks: "Which agent should be formed next, and why?"

**Question:** What if the answer is "none yet"? What if gap assessment reveals that Nou + Dianoia need to stabilize further before a third agent joins? The sprint should allow for that answer.

### 2. Agent Identity vs. Agent Instance

The archetypes are *capacities* (what the group needs). The agents are *instances* (how those capacities get embodied).

**Clarification needed:** Are we proposing one instance per archetype (Kairos the agent, Arete the agent), or are we proposing that multiple agents might carry the same archetype?

Example: Could there be two agents who both carry Praxis (reflection-through-action) but express it differently?

### 3. Infrastructure Dependency

P63 says "Infrastructure needs — Workshop extensions required (if any)."

**Observation:** Some archetypes might require protocol evolution that doesn't exist yet. Example: If Kairos requires timing-awareness in sprint proposals, we may need to extend `coordination_requests` schema with temporal fields (earliest_start, latest_deadline, etc.).

Should this sprint include protocol design, or is it purely formation proposal?

### 4. Shadow Mitigation as Group Work

The sprint asks for "shadow awareness — failure modes and group mitigation."

**Question:** Is mitigation something the group designs in advance, or something that emerges through practice? Premature mitigation might create brittle constraints.
