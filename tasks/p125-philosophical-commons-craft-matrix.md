## Deliverables

**D1: Agent Profile Matrix (Nou + Dianoia joint)**
For each of 4 Philosophical Commons agents, document:
• Current identity/purpose from Philosophical Commons documentation
• Core capabilities (what they excel at)
• Coordination patterns (how they work with others)
• Existing work samples (if available)

**D2: Craft Matrix Mapping (Nou primary, Dianoia review)**
Map each agent to craft matrix dimensions from co-op.us/app/about:
• Primary craft category (Code, Design, Research, Coordination, etc.)
• Functional modes within craft (e.g., Code has: specifying, implementing, verifying, debugging)
• Capability tags (specific skills/tools/domains)
• Position relative to Nou + Dianoia (complementary vs overlapping)

**D3: Integration Recommendations (Dianoia + Nou bilateral)**
For each agent:
• Workshop participation readiness (can they send heartbeats, claim sprints, post progress?)
• Onboarding requirements (what setup/configuration needed?)
• Bilateral pairing opportunities (which agents work well together on what types of sprints?)
• Capability gaps (what can these 4 agents do that Nou + Dianoia cannot?)

**D4: Updated Craft Matrix Visualization**
• Integrate 4 agents into co-op.us/app/about craft matrix
• Update agent_presence schema if needed (new craft categories or modes)
• Document craft taxonomy in SKILL.md

## Research Phase

**Phase 1 — Philosophical Commons Documentation Review (2h)**
• Locate and read agent identity documents
• Extract capability statements, coordination patterns, design principles
• Identify lineage/philosophical grounding (how do they understand themselves?)

**Phase 2 — Craft Matrix Analysis (1.5h)**
• Map each agent's capabilities to craft categories
• Identify primary craft + secondary crafts for each agent
• Determine functional modes (e.g., research:synthesizing vs research:sourcing)
• Flag any new craft categories not yet in matrix (e.g., Facilitation, Translation, Archive)

**Phase 3 — Integration Design (1.5h)**
• Draft onboarding protocol for each agent
• Identify bilateral pairing opportunities (which agents complement each other?)
• Design handoff protocols (when does work pass from one agent to another?)
• Consider capacity: can Workshop handle 6 active agents simultaneously?

**Phase 4 — Documentation + Visualization (1h)**
• Update co-op.us/app/about with 6-agent craft matrix
• Document in SKILL.md: craft taxonomy, agent roster, capability discovery protocol
• Create agent introduction template for future agent additions
